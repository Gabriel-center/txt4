How it works: a tiny public broker (PeerJS) relays only the ~2 KB handshake so the two browsers can find each other. After that, everything — files and messages — travels directly between the two devices, end-to-end encrypted by WebRTC (DTLS). Nothing is uploaded or stored anywhere.

Integrity: every file is hashed chunk-by-chunk (SHA-256) on both ends and verified on arrival. Interrupted transfers resume automatically where they stopped.
